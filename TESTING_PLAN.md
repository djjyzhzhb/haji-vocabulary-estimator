# 词汇估算系统 测试方案

## 📋 概述

本文档描述了词汇估算系统的完整测试策略，包括单元测试、集成测试和端到端测试。

## 🎯 测试目标

- 确保计算引擎的正确性和准确性
- 验证所有UI组件的正常渲染和交互
- 测试路由导航功能
- 确保状态管理（Zustand）的正确性
- 验证图表可视化的正确性
- 测试响应式设计

## 🛠️ 测试工具

- **单元/集成测试**: Vitest + React Testing Library
- **E2E测试**: Playwright
- **代码覆盖率**: V8 Coverage (Vitest内置)

## 🚀 快速开始

### 安装依赖

```bash
npm install
```

### 运行测试

#### 单元测试
```bash
# 运行所有单元测试
npm run test

# 监听模式
npm run test:watch

# 查看覆盖率
npm run test:coverage
```

#### E2E测试
```bash
# 首次使用需要安装浏览器
npx playwright install

# 运行E2E测试
npm run test:e2e

# UI模式运行
npm run test:e2e:ui
```

## 🔧 测试最佳实践

1. **AAA模式**: Arrange（准备）- Act（执行）- Assert（断言）
2. **一个测试一个断言**: 每个测试只验证一个行为
3. **测试独立**: 测试之间不共享状态
4. **描述清晰**: 测试名称清楚表达测试意图
5. **快速反馈**: 测试执行时间要短

## 📞 故障排除

### 测试超时
- 检查组件是否有异步操作
- 增加等待时间或简化测试

### 覆盖率不足
- 添加更多边界情况测试
- 测试错误处理路径

### E2E测试失败
- 确保本地开发服务器正在运行
- 检查选择器是否正确
